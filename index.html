Deployment Steps

Follow these 6 steps to complete the integration. Commands below are updated with your config.

1

Install Blobfuse2 & FUSE

First, we need to install the Microsoft package repository and the necessary FUSE libraries on your Linux host. FUSE (Filesystem in Userspace) allows non-privileged users to create file systems.

wget https://packages.microsoft.com/config/ubuntu/20.04/packages-microsoft-prod.deb
sudo dpkg -i packages-microsoft-prod.deb
sudo apt-get update
sudo apt-get install libfuse3-dev fuse3 blobfuse2 -y
2

Create Configuration File

Create the configuration YAML file. This tells Blobfuse how to connect to Azure and how to handle caching.

sudo touch /etc/blobfuse2-config.yml
sudo nano /etc/blobfuse2-config.yml

# PASTE THIS CONTENT:
allow-other: true

logging:
  type: syslog
  level: log_debug

components:
  - libfuse
  - file_cache
  - attr_cache
  - azstorage

libfuse:
  attribute-expiration-sec: 120
  entry-expiration-sec: 120
  negative-entry-expiration-sec: 240

file_cache:
  path: /mnt/blobfusetmp
  timeout-sec: 120
  max-size-mb: 4096

azstorage:
  type: block
  account-name: [ENTER ACCOUNT NAME]
  account-key: [ENTER KEY]
  endpoint: https://[ACCOUNT].blob.core.windows.net
  mode: key
  container: [ENTER CONTAINER]

Understanding Cache Config

Blobfuse caches files locally (`/mnt/blobfusetmp`) to speed up access. `timeout-sec` determines how long a file stays in cache before re-validating with Azure.

Higher timeout = Faster Read / Lower Consistency
3

Secure the Credentials

Critical Security The config file contains your Storage Key. We must restrict access to the root user only. 

sudo chown root:root /etc/blobfuse2-config.yml
sudo chmod 600 /etc/blobfuse2-config.yml
4

Create Mount Points

Create the destination directory for your files and the temporary cache directory.

sudo mkdir -p /mnt/data
sudo mkdir -p /mnt/blobfusetmp
sudo chown root:root /mnt/blobfusetmp
5

Mount Storage

Execute the mount command using the config file created in Step 2.

sudo blobfuse2 mount /mnt/data --config-file=/etc/blobfuse2-config.yml
Verification: Run df -h after this command. You should see a line entry for blobfuse2 mounted at /mnt/data.
6

Run Docker Container

Finally, start your Docker container and bind the host mount point (`/mnt/data`) to a path inside the container.

sudo docker run -it -d \
  --name my-app-container \
  -v /mnt/data:/app/data:shared \
  -e Account_Name="[ENTER ACCOUNT NAME]" \
  your-docker-image:latest
Key Flag: :shared
This flag is crucial for FUSE mounts. It ensures that the mount created on the host is correctly propagated and visible inside the Docker container.

Unmounting & Cleanup

Need to disconnect the storage safely? Use the following command. Do not simply delete the folder while it is mounted.

sudo blobfuse2 unmount /mnt/data
